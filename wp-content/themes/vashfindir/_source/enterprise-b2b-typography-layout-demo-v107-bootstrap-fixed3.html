Кнопки: CTA и статусы

Минимализм + доверие: один “основной” CTA, спокойные вторичные действия, красный — только риск.

Цветовая логика для Enterprise B2B (финансы): Blue = действие, Navy = структура, Red = риск.
Примеры
CTA fill: --cta-secondary CTA ref: --cta Brand: --brand
Primary CTA
Регистрация / Оплатить / Начать — синий fill (в финтехе воспринимается как “контроль”).
Secondary
Войти / Подробнее / Сравнить — outline, без лишнего давления.
Danger
Удалить / критические риски — только красный. Не использовать как маркетинговый акцент.

Шкала заголовков и текстов

Цель — минимализм, читаемость и “дорогая” уверенность. Без лишней декоративности.

Основной шрифт (body): system-ui, -apple-system, "Segoe UI", Roboto, Arial, sans-serif
Токен: --font-sans → body { font-family: var(--font-sans); }

H1 — Заголовок

Размер: clamp(32px, 3.2vw, 46px), вес: 900

H2 — Подзаголовок

Размер: clamp(24px, 2.1vw, 34px), вес: 900

H3 — Заголовок карточки

Размер: 20px, вес: 900
Body — обычный текст
Размер: 16px, line-height: 1.55
Small / helper
Размер: 13px, line-height: 1.45

Лучшие практики (коротко)

  • 1 смысл = 1 акцент (CTA один цвет).
  • Текст ограничиваем до 56–68ch.
  • Отступы секций: clamp(48px..80px).
  • Весов немного: 400 / 600 / 900.
  • Карточки — единый padding: 16–20px.

Требования к фото для шаблона

Нормы под наши токены: --content-max: 1140px, радиусы 6px, фон секций --bg-soft. Фото — часть доверия, а не декор.

Рекомендуемые форматы

  • Портрет (основатель/команда): 4:5 (основной), альтернативно 1:1 для аватаров.
  • Обложка (кейсы/блоки): 16:9.
  • Формат файла: предпочтительно .webp, допустимо .jpg.

Минимальные размеры (px)

  • Mobile: 768px по меньшей стороне.
  • Tablet: 1200px по меньшей стороне.
  • Desktop: 1600px по меньшей стороне (ключевые фото).

Свет и цвет

  • Свет мягкий, равномерный, без жёстких теней на лице.
  • Цвет натуральный, без кислотной сатурации.
  • Запрещено: градиенты/затемнения/слои поверх лица, “glass”, неон.

Кадрирование

  • Без обрезки головы/подбородка, не “впритык”.
  • Глаза примерно на верхней трети кадра.
  • Фон: однотонный, тёмно‑синий/графит/светло‑серый.
Как ведёт себя на экранах
Desktop: 2 карточки в ряд (col-lg-6), высота фото задаётся ratio.
Tablet: 1 карточка в ряд по умолчанию (col-12), можно включать 2× на md.
Mobile: 1 карточка в ряд, без “гигантских” вертикальных отступов.
Технически
Используйте <picture> + srcset (1x/2x), задавайте alt. Внутри блока — никаких плашек поверх лица.

Берем ответственность за управленческие финансы: учет, планирование, алерты, решения и регулярный ритм “до”.

ДДС и платежный календарь ОПиУ и маржинальность Сценарии и решения
Что меняется у клиента
  • Понятно, где деньги и что будет через 2–6 недель.
  • Риски подсвечиваются “светофором” до кассового разрыва.
  • Решения принимаются на цифрах: маржа, расходы, долг, сценарии.
Для кого наши продукты
  • Малый и средний бизнес с регулярными платежами и сезонностью.
  • E-commerce, производство, услуги — где важны оборотка и маржа.
  • Выручка от 2 млн/мес и выше (ориентир).
Кому подойдет быстрее всего
  • Если “денег вроде много”, но постоянно не хватает на платежи.
  • Если нет ясной маржинальности по продуктам/каналам.
  • Если решения принимаются “на ощущениях”.
