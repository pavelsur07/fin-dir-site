# ✅ designer-checklist.md
Проект: Ваш ФинДир  
Статус: обязательный чек-лист перед сдачей дизайна

---

## 1. Общая проверка

- [ ] Использованы **тексты строго из md-файлов**
- [ ] Структура страниц соответствует wireframe
- [ ] Нет добавленных «маркетинговых» элементов
- [ ] Нет стоковых фото

---

## 2. Hero-блоки

### Главная (`home`)
- [ ] Фото основателя (реальное)
- [ ] Нет ноутбуков / графиков / экранов
- [ ] Заголовок читается с первого взгляда
- [ ] CTA виден без скролла

### `service-findir`
- [ ] Фото основатель + команда
- [ ] Диалог, не позирование
- [ ] Нет ощущения «корпоративного стока»

---

## 3. Кнопки и CTA

- [ ] На экране не больше 1 primary CTA
- [ ] Secondary CTA визуально слабее primary
- [ ] Нет слов «купить», «заказать», «оформить»

---

## 4. Иконки

- [ ] Используются только там, где разрешено
- [ ] Один стиль и толщина линии
- [ ] Нет цветных / иллюстративных иконок

---

## 5. Тарифы

- [ ] Нет иконок и фото
- [ ] Чёткая визуальная иерархия
- [ ] Цена читается быстро
- [ ] Нет фич-листов

---

## 6. Визуальный шум

- [ ] Достаточно воздуха между блоками
- [ ] Нет перегруженных экранов
- [ ] Один экран = одна мысль

---

## 7. Адаптив

- [ ] Корректно на мобильных
- [ ] CTA доступны и читаемы
- [ ] Текст не ломается

---

## 8. Финальный вопрос (обязательный)

Ответь «да» на все пункты:
- [ ] Страница выглядит спокойно
- [ ] Нет ощущения типового SaaS
- [ ] Видно, что за продуктом стоят люди
- [ ] Хочется доверять

Если хотя бы один ответ «нет» — работа не готова.

---

## 9. Статус

`designer-checklist.md` — утверждён.  
Используется для приёмки дизайна.
